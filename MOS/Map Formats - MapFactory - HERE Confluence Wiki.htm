Map Formats

Skip to end of metadata
  • Created by Seheda, Ihor, last modified on Apr 17, 2015
Go to start of metadata

Map Containers

The map is contained in a .cdt file. The container is optimized for server usage and batch map processing.

Formats

The map is available in the following formats:

  • 6, for client ID 1.3 and 1.4, delivered as a single CDT file and map loader packages.
  • 8, for client ID 1.7, delivered as a single CDT file and new diskcache packages. See also ChangelogFormat8.

Format is made up of multiple layers. Format Matrix describes each format and layer.

Each layer has a specific binary format:

  • Binary formats introduced in format 5 (now deprecated, but some layer formats are still in use)
  • Binary formats introduced in format 6
  • Binary formats introduced in format 8
  • NewBinaryFormats under development and investigation

Special Encodings are also provided for specific types and structures.

About format 8

With "Format 8" we name the map format developed for the beginning of 2012, it is our major format with the following main features:

  • Proper support for Data Groups features of the DiskCache
  • Disputed Territories attributes and Political Views
  • Natural Guidance
  • Phonetic Names
  • General improvements for routing
  • General improvements for rendering
  • Contraction Hierarchies
  • Compile-time Routing Graph
  • Support for 3D Globe projections
  • ...

The format is designed with future compatibility and extensibility since the first day. In this way we hope to delay another format break as long as we can.

Map #47 is the first available in such format, together with older formats, and is the first map releases to Microsoft for the integration of MOS in WP8 "Apollo". Format 8 will also be used in "Apollo+", with probably some layer added.

Format 8 is stable, but can be extended preserving backward compatibility. Please have a look at the Ñhangelog for further information.
